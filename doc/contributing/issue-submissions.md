# Issue Submission Guide

All development tasks, bug reports, and feature requests must be submitted as issues before any work begins. This ensures transparency, proper tracking, and team alignment.

## Contents

- [Steps to Submit an Issue](#steps-to-submit-an-issue)
- [Why Use Issues?](#why-use-issues)

---

## Steps to Submit an Issue

1. Navigate to the repository's Issues tab.
2. Click "New Issue" and select the appropriate template (Bug Report, Development Task, etc.).
3. Fill out all required sections:
   - Provide a clear and concise title.
   - Describe the issue or task in detail.
   - Include steps to reproduce (for bugs) or acceptance criteria (for tasks).
   - Add any relevant screenshots, environment details, or additional context.
4. Submit the issue and reference its number in any related pull requests.

---

## Why Use Issues?

- Ensures all work is documented and visible.
- Facilitates discussion and review before implementation.
- Links code changes directly to tracked tasks.

**Reminder:** No development work should begin without a corresponding issue.
